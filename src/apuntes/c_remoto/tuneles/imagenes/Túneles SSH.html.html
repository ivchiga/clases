Ejemplo 1

Caso 2

Necesitamos acceder a un servicio de la red remota y este servicio no es accesible directamente al no haber ningún puerto de la IP pública redireccionado al mismo. En este caso tenemos un servidor web solamente accesible desde la red local en la IP 192.168.10.21.

En este caso, la máquina remota ejecuta tanto el servidor SSH como el servidor MySQL (puerto por defecto 3306). Según esto, la orden adecuada para la creación el túnel es:

$ ssh -N -L 8080:192.168.10.21:80 omar@servidor.midominio.com

Descripción

-N

    indica que no se debe iniciar una terminal tras la conexión

-L

    suministra los parámetros para el túnel

8080

    es el puerto en la máquina local encaminado al servidor web

192.168.10.21

    es la dirección de la máquina a contactar desde el punto de vista del servidor ssh y como el servidor web está ejecutándose en otra máquina de la red ponemos su dirección

80

   es el puerto que atiende el servidor web

A partir de este momento, se puede acceder al servidor web apuntando al puerto 8080 de la máquina local (localhost o 127.0.0.1).

Acceso al servicio web remoto
introducimos en navegador URL http://localhost:8080
Ejemplo 2

Caso 2

Necesitamos acceder a un servicio de la máquina remota y este servicio no está configurado para prestarse sobre una conexión cifrada o simplemente está configurado para atender peticiones locales solamente. Un ejemplo de esto es el servidor de bases de datos MySQL que normalmente atiende conexiones que se gestionan desde la propia computadora, rechazando a las procedentes de red.

En este caso, la máquina remota ejecuta tanto el servidor SSH como el servidor MySQL (puerto por defecto 3306).

$ ssh -N -p 22 -L 13306:localhost:3306 omar@servidor.midominio.com

Descripción

-p 2222

     es el puerto al que escucha el demonio ssh de servidor.midominio.com (si no se especifica escucha por el puerto por defecto, 22)

-N

    indica que no se debe iniciar una terminal tras la conexión

-L

    suministra los parámetros para el túnel

13306

    es el puerto en la máquina local encaminado al servidor MySQL

localhost

    es la dirección de la máquina a contactar desde el punto de vista del servidor ssh y como el servidor MySQL está ejecutándose en el propio servidor.dominio.com, se usa localhost para autoreferirse

3306

   es el puerto que atiende el servidor MySQL

A partir de este momento, se puede usar el programa de administración de bases de datos apuntando al puerto 13306 de la máquina local (localhost o 127.0.0.1) para acceder al servidor remoto.

Acceso al servicio
$ mysql -h localhost -p 13306
